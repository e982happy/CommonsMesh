# 防女巫攻击与防篡改验证设计 (Sybil Resistance & Anti-Tampering)

CommonsMesh 作为一个本地优先、去中心化的社区互助网络，在开放环境中面临严峻的身份滥用和数据篡改威胁。为了保障社区资源的有效匹配与决策的公正性，本协议引入了多层防御机制。

## 1. 防女巫攻击 (Sybil Resistance)

女巫攻击是指恶意用户通过创建大量虚假身份来控制网络、操纵投票或耗尽社区资源。在没有中心化权威机构的 P2P 网络中，我们采用以下混合策略来提高创建虚假身份的成本。

### 1.1 设备与应用证明 (Attestation)

所有消息必须携带设备或应用的真实性证明。协议支持多种级别的证明：
- **Software Attestation**: 验证应用构建哈希（App Build Hash）和清单哈希（Manifest Hash），确保消息来自官方发布的客户端，而非自动化脚本。
- **Hardware Attestation**: 利用移动设备的硬件安全模块（如 iOS Secure Enclave 或 Android Keystore）生成设备唯一标识，并签名证明密钥由受信任的硬件生成。这极大地提高了批量创建虚假节点的成本，因为每个节点需要真实的物理设备。
- **Social Attestation**: 基于社区信任图谱。新用户必须获得现有高信任度成员的邀请或背书才能获得基础信任分。

### 1.2 动态信任评分系统 (Dynamic Trust Scoring)

单纯的身份验证不足以防止恶意行为，协议内嵌了动态的信任评分机制：
- **初始受限**: 新加入的节点信任分为 0，其发布的动议或需求在全网的可见度和优先级受到限制。
- **行为累加**: 用户通过完成任务、参与投票、提供资源等建设性行为，可获得其他节点的正面评价，从而缓慢增加信任分。
- **衰减与惩罚**: 信任分会随时间自然衰减（Proof of Active Participation）。若用户被举报并经社区仲裁确认恶意行为，信任分将被大幅扣减甚至清零。
- **资源匹配阈值**: LLM 匹配引擎在生成合作动议时，可设置最低信任分阈值，自动过滤低信誉的潜在参与者。

### 1.3 工作量与稀缺资源证明

在某些高价值场景（如选举投票、重大项目立项）中，协议可要求附带计算量证明（PoW）或质押某种社区内部的稀缺资源（如积分、代币），进一步增加女巫攻击的经济成本。

## 2. 防篡改验证 (Anti-Tampering)

为了确保分布式图谱数据库中事件日志的不可变性和因果一致性，协议在核心验证层实施了严格的密码学约束。

### 2.1 规范化 JSON 与签名

所有消息在计算哈希和签名之前，必须经过严格的 JSON 规范化（Canonical JSON）处理。这确保了无论对象键的顺序如何，同一消息在不同节点上的哈希值绝对一致。
- **Payload Hash**: 仅对消息的主体内容进行哈希，确保内容未被修改。
- **Envelope Signature**: 使用发送者的 Ed25519 私钥对包含元数据（时间戳、发送者信息、权限凭证）和 Payload Hash 的整个信封进行签名，保证消息的不可否认性。

### 2.2 严格的哈希链 (Hash Chain)

CommonsMesh 采用追加写入（Append-Only）的事件日志模型。
- **前驱链接**: 每一条新消息的 `authz.prev_event_hash` 必须严格指向本地日志中上一条消息的完整 `chain_hash`。
- **因果一致性**: 这种设计将所有事件串联成一条单向哈希链。任何对历史消息的篡改都会导致后续所有消息的哈希验证失败，从而被网络中的其他节点立即拒绝。

### 2.3 防重放与时间窗口

恶意节点可能会拦截并重新发送历史上的合法消息（重放攻击）。
- **Nonce 缓存**: 验证器维护每个用户的 Nonce 集合，拒绝处理具有相同 Nonce 的消息。
- **时间窗口验证**: 消息必须在 `valid_from` 和 `valid_to` 定义的严格时间窗口内被处理。结合本地系统时间检查，防止攻击者囤积消息并在未来恶意重放。

### 2.4 能力令牌 (Capability Token) 强制验证

权限控制基于签名的能力令牌。
- **令牌签名验证**: 核心引擎在验证消息权限时，必须首先验证能力令牌本身的签名是否来自合法的发行者。
- **范围与次数约束**: 严格检查令牌允许的操作类型（如 `propose`, `vote`）、作用域（特定项目或主题）以及剩余使用次数（`max_count`），防止越权操作。

通过上述机制的协同工作，CommonsMesh 能够在去信任的网络环境中构建出一个安全、可靠且具有韧性的社区协作基础设施。
